# 整体功能设计(定稿 2026-07-04)

产品一句话:**把你自己订阅的信息源和书,每天自动变成跑步/通勤时能连续收听的私人电台。**
定位是"信息获取的音频界面"(效率工具),不是"AI 播客节目"(内容)。市场依据见 [PRODUCT.md](./PRODUCT.md)。
协议:AGPL-3.0(个人自托管完全自由;拿去做闭源托管服务必须公开改动)。

## 架构:两个面,零后台服务

```
┌─ 生成端(无界面"后台")──────────────┐
│ 定时管线:抓源 → LLM 对话稿 → TTS     │  跑在:自己的机器 systemd /
│ 全量书籍转换:books:sync              │  fork 后的 GitHub Actions
└──────────────┬───────────────────────┘
               │ 只写静态文件
               ▼
┌─ 存储/分发(Supabase Storage)───────┐
│ feed.xml + books.json + mp3          │ ← 任何播客 App 直接订 feed.xml
└──────────────┬───────────────────────┘
               │ 只读
               ▼
┌─ 播放前台(纯静态页,Cloudflare Pages)┐
│ 资讯 / 书籍 / 设置                    │  无服务端代码
└───────────────────────────────────────┘
```

刻意不做常驻后台服务:运行时没有服务器、没有数据库、没有 API。由此得到三个开源产品的关键性质——免费额度内永久跑、自托管者零运维、没有会半夜挂掉的东西。管理界面以播放器"设置页"的形式长在前台(直写 bucket 配置),不另起系统。

## 用户分层与门槛策略

| 层 | 用户 | 入口 | 门槛 |
|---|---|---|---|
| L1 极客(v1 目标) | 会 fork、填 secrets | 开源仓库 + GitHub Actions 模板 | 首次搭建 ~15 分钟,他们不觉得是门槛 |
| L2 轻度技术用户(v1.5) | 会注册网站、不碰命令行 | "Deploy to Railway/Render" 一键按钮 | 填 3 个 key |
| L3 小白(托管版,远期) | 只会粘贴链接 | 收费托管:注册→贴 RSS→得订阅地址 | 零 |

原则:**不把自托管版磨成小白产品**——首次搭建的 key 申请环节界面救不了,小白的答案是托管版"收费帮你省事"(Plausible/Cal.com 分层)。听众侧永远零门槛:一套部署,家人朋友打开网页/订阅播客即听。

## 功能清单

状态:✅ 已上线 / 🔨 v1(开源发布前完成)/ 🧊 v2(有真实外部用户后)

### 1. 源
- ✅ 任意 RSS/Atom 源;Reddit 429 重试;按源轮询防单源垄断
- 🔨 单篇 URL 即时转一集(CLI + bucket 队列,补"稍后读清空"场景)
- 🔨 源管理设置页(加删源不再改 JSON)
- 🔨 OPML 导入
- 🔨 公众号/知乎接入指南(自托管 RSSHub / wewe-rss,文档)
- 🧊 Newsletter 邮箱转发入源

### 2. 生成
- ✅ 对话式双主播(A 主持×B 嘉宾),600-900 字硬限防注水;解析失败退单人
- ✅ 直读模式;书籍永远直读;每集尾部报来源;英文标题不翻译
- ✅ 定时自动生成(systemd timer)
- 🔨 GitHub Actions 模板(fork + secrets = 无服务器日更,自托管门槛从"有 Linux 机器"降到"有 GitHub 账号")
- 🔨 每源可选形态(新闻对话/长文直读)
- 🧊 TTS 多实现:MiniMax(商用)、本地 Kokoro(吸引本地派)
- 🧊 按跑步时长点单("今天 40 分钟"自动凑节目单)

### 3. 书
- ✅ txt 整本预生成:断点续传、逐集可听、删 txt 即释放空间
- 🔨 epub 直接支持(丢进 books/ 自动转;mobi 用 Calibre 转 epub,文档说明)
- 🔨 对话导读集(每本书第 0 集:15 分钟"两人聊这本书",NotebookLM 只能导读且长书截断,我们导读+全本都有)
- 🧊 章节感知切分(集名用章节名)

### 4. 听
- ✅ Web 播放器:资讯/书籍 tab、倍速、睡眠定时、锁屏控制、跨集连播、书籍断点记忆、错误原因上屏
- ✅ 标准播客 feed(任何播客 App 可订——播放器基础能力"白嫖"成熟 App,Huxe 死于自建这层)
- 🔨 回退/快进 15s(跑步走神重听是高频动作,生死线)
- 🔨 feed iTunes 元数据补齐(封面/分类,验证 Apple Podcasts 全流程)
- 🔨 PWA + 离线缓存(出门前自动缓存今日集,无信号可听)
- 🧊 完听率自统计(反哺选源)

### 5. 自托管体验
- ✅ .env.example;历史无密钥、无版权文本
- 🔨 README 重写(中英双语,"5 分钟跑起来")
- 🧊 Docker 镜像(有人要了再做);Deploy-to-Railway 按钮(v1.5)

## 明确不做(负空间)

- ❌ 自建移动 App(Huxe 之死;播客 App 生态已解决)
- ❌ chat/问答入口(Snipd/Curio 教训:打开即播)
- ❌ 公开内容平台/推荐流(来福之死:泛内容 AI 电台已证伪)
- ❌ 播客创作者工具(Jellypod/Wondercraft 红海)
- ⏸ 多声音选择(用户已定暂缓)

## v1 完成定义

> 一个陌生开发者 fork 仓库、填 3 个 secret、扔一个 OPML 和一本 epub,第二天早上他的播客 App 里有 5 集对话资讯和一本书的导读——全程不碰服务器。

做到这个,才发 Show HN 和知乎。

v1 顺序:回退 15s → feed 元数据 → epub 转换 → 单篇 URL → 对话导读 → PWA 离线 → 源管理页 → GHA 模板 → README 重写。
