它做什么

📰 RSS → 对谈播客

你订阅的 RSS、Reddit、任意文章链接,LLM 改写成两位主播的自然对谈,TTS 合成音频,每天早上自动出现在你的播客 App。

📚 电子书 → 有声书

epub/pdf/txt/html 上传即转,整本连载 + 一集"两位主播聊这本书"的导读;英文书可整本翻译成中文再听。上传几分钟后就能边生成边听。

📡 标准播客 RSS

Apple Podcasts、小宇宙、Pocket Casts……任何播客 App"通过 URL 关注"直接订阅,不锁定任何平台。

🏃 跑步友好播放器

PWA 网页播放器:锁屏控制、±15 秒、倍速、睡眠定时、离线缓存、断点续听,出门前打开一次即可离线听。

🧱 零后台架构

批处理管线 + 对象存储 + 静态页,全部跑在 Cloudflare 免费额度(R2 10GB、出口流量免费)。没有要维护的服务器。

🔓 开源 AGPL-3.0

代码完全开放,数据全在你自己的账号里。LLM 用任意 OpenAI 兼容端点,不填也能退化为原文直读。

三步跑起来(自部署)

克隆并安装

git clone 仓库,npm install,在 .env 里填 Cloudflare 的两个值(注释里有指路)。

一键部署

npm run setup 自动建存储、开公开域名、生成管理口令、部署播放器,最后打印你的播放器地址和播客订阅地址。

开始听

npm run pipeline 生成第一批;配上 GitHub Actions 或 cron,以后每天自动更新。手机上传电子书,几分钟后边生成边听。

不想自己部署?托管版排队中

我们在评估面向普通用户的托管版(注册即用、无需任何技术操作)。留下邮箱,上线第一时间通知你;排队人数直接决定我们做不做、多快做。

常见问题

和"AI 播客"产品(Recast、PlayNote)有什么区别?

那批产品替用户承担 TTS 成本、做内容平台,大多已关停。本项目是"信息获取的音频界面":内容是你自己的订阅和书,成本跑在你自己的免费额度里,所以可以一直活着。

RSS 转播客的音质怎么样?

默认微软 Edge 神经网络声音(个人自用免费),中文自然度接近真人播报;资讯是双主播对谈形式,不是机械朗读。商用场景可替换为任何 TTS 服务(留有接口)。

epub 转有声书要多久?

生成速度约为播放速度的 10 倍:一本 20 万字的书全部生成约 2~3 小时,但上传几分钟后第 1 集就能听,边生成边听,追不上它。

英文书能用中文听吗?

能。内置整本翻译命令,英文 epub 翻成中文后再生成中文有声书;不翻译则自动切换英文声音朗读。

需要花多少钱?

存储和流量:Cloudflare 免费额度内(R2 10GB)。LLM 改写:按用量计费,每天 8 集资讯大约几毛钱;不配 LLM 也能用(原文直读)。TTS:个人自用免费。
